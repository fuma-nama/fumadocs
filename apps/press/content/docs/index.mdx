## What is Fumapress?

Fumapress is a site generator powered by [Fumadocs](https://fumadocs.dev) & [Waku](https://waku.gg), as compared to Fumadocs, it manages more features such as routing, while being more opinionated.

It is perfect if you want a beautiful, flexible docs, without revealing too much complexity.

Since the app is still a Waku.js app, you can easily transition to a full Fumadocs app without heavy migration.

## Getting Started

Install the dependencies:

```npm
npm i waku fumapress fumadocs-ui fumadocs-core fumadocs-mdx react react-dom react-server-dom-webpack

npm i tailwindcss @tailwindcss/vite @types/react @types/react-dom typescript vite -D
```

Create the config files:

```ts tab="waku.config.ts"
import { defineConfig } from 'waku/config';
import tailwindcss from '@tailwindcss/vite';
import press from 'fumapress/vite';
import mdx from 'fumadocs-mdx/vite';

// the config file for Waku & Vite
export default defineConfig({
  vite: {
    plugins: [press(), mdx(), tailwindcss()],
  },
});
```

```ts tab="source.config.ts"
import { defineDocs } from 'fumadocs-mdx/config';
import { metaSchema, pageSchema } from 'fumadocs-core/source/schema';

// the config file for Fumadocs MDX, see https://fumadocs.dev/docs/mdx
export const docs = defineDocs({
  dir: 'content',
  docs: {
    async: true,
    schema: pageSchema,
    postprocess: {
      includeProcessedMarkdown: true,
    },
  },
  meta: {
    schema: metaSchema,
  },
});
```

```ts tab="press.config.ts"
import { defineConfig } from 'fumapress';
import { llmsPlugin } from 'fumapress/plugins/llms.txt';
import { takumiPlugin } from 'fumapress/plugins/takumi';
import { flexsearchPlugin } from 'fumapress/plugins/flexsearch';
import { loader } from 'fumadocs-core/source';
// don't worry if this file is missing, we will run the dev command later to generate this file
import { docs } from './.source/server';
import { fumadocsMdx } from 'fumapress/adapters/mdx';

export default defineConfig({
  loader: loader(docs.toFumadocsSource(), {
    baseUrl: '/',
  }),
  site: {
    name: 'Fumapress',
  },
  // extend via plugins
  plugins: [flexsearchPlugin(), llmsPlugin(), takumiPlugin()],
  // use different content sources
  adapters: [fumadocsMdx()],
});
```

Add scripts:

```json title="package.json"
{
  "scripts": {
    // [!code ++:4]
    "dev": "waku dev",
    "build": "waku build",
    "start": "waku start",
    "types:check": "fumadocs-mdx && tsc --noEmit"
  }
}
```

Start the dev server:

```npm
npm run dev
```

You can now start writing Markdown files under the `content` directory.

```mdx title="content/index.mdx"
---
title: My Page
description: Hello World
---

# Overview

This is my first document.
```

### Custom Styles

Create a `src/app.css` file:

```css title="src/app.css"
@import 'tailwindcss';
@import 'fumadocs-ui/css/neutral.css';
@import 'fumadocs-ui/css/preset.css';
@import 'fumapress/css/preset.css';

/* add your styles below */
```

It will be automatically loaded, you can restart the dev server if there's one running.

### Custom Routing & UI

The routing config is optional, you can create one if needed.

```tsx title="src/waku.server.tsx"
import adapter from 'waku/adapters/default';
// your press.config.ts file
import pressConfig from '../press.config';
import { createRouter } from 'fumapress/router';
import { createDocsLayout } from 'fumapress/layouts/docs';

const router = createRouter(pressConfig, {
  // customize layout for pages
  page: createDocsLayout(),
});

export default adapter(router.createPages());
```

See the [Waku Config-based Routing](https://waku.gg/#routing) section if you want to create custom routes.

```tsx title="src/waku.server.tsx"
import adapter from 'waku/adapters/default';

const pages = router.extend(async ({ createPage }) => [
  // note: root element is already created by Fumapress
  createPage({
    render: 'dynamic',
    path: '/hello-world',
  }),
]);

export default adapter(pages);
```

### Render Mode

By default, Fumapress build both static & dynamic routes. If you plan to deploy it to a CDN as a static site, you need to specify a render mode.

```ts title="press.config.ts"
import { defineConfig } from 'fumapress';

export default defineConfig({
  // ...
  // [!code ++]
  renderMode: 'static',
});
```

This will force Fumapress to build all routes as static.

### Internationalization

To configure internationalization, define an i18n config:

```ts title="press.config.ts"
import { coreI18n, defineConfig, type I18nConfig } from 'fumapress';
import { loader } from 'fumadocs-core/source';
import { docs } from './.source/server';

// [!code ++:14]
const i18n: I18nConfig = {
  languages: {
    cn: {
      displayName: 'Chinese',
      translations: {
        // add your translations
      },
    },
    en: {
      displayName: 'English',
    },
  },
  defaultLanguage: 'en',
};

export default defineConfig({
  loader: loader(docs.toFumadocsSource(), {
    baseUrl: '/',
    // [!code ++]
    i18n: coreI18n(i18n),
  }),

  // [!code ++]
  i18n,
});
```

Add Markdown/JSON files for different languages by attending `.{locale}` to your file name, like:

<Files>
  <Folder name="content" defaultOpen>
    <File name="meta.json" />
    <File name="meta.cn.json" />
    <File name="get-started.mdx" />
    <File name="get-started.cn.mdx" />
  </Folder>
</Files>

### Meta

To add meta tags, you can leverage the `meta` config.

Before specifying it, it's better to rename your `press.config.ts` to `press.config.tsx` for JSX syntax.

```tsx title="press.config.tsx"
import { defineConfig } from 'fumapress';

export default defineConfig({
  // [!code ++:16]
  meta: {
    root() {
      return (
        <>
          <meta property="og:type" content="website" />
          {/* you can use it for link tags as well */}
          <link rel="preconnect" href="https://fonts.googleapis.com" />
          <link rel="preconnect" href="https://fonts.gstatic.com" crossOrigin="" />
          <link
            href="https://fonts.googleapis.com/css2?family=Geist+Mono:wght@100..900&family=Geist:wght@100..900&display=swap"
            rel="stylesheet"
          />
        </>
      );
    },
  },
});
```
